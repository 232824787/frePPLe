Modeling limitations

The model components are very generic and can easily be combined and nested into a wide range of models. Not all of these constructs are handled correctly by the solver, either because of some implicit assumption, either because of the code structure, either because of uncomplete code, etc... In this section we document a number of such cases:

  • Operation hierarchies should be restricted to two levels only, where possible.
  • Attach flows and loads to the lowest operations only, where possible.
  • Don't attach a consuming flow at multiple levels in an operation hierarchy.
    E.g. Top operation consumes a certain material, and a suboperation consumes the same material again.
  • Similarly, don't load the same resource at multiple levels in an operation hierarchy
  • Hierarchy structures of resources and buffers are untested. Only demand hierarchies are currently supported.
