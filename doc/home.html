FREPPLE
a Free Production Planning Library

Production planning problems are very common in manufacturing and distribution. System support is available from a wide range of vendors. All tools on the market have pretty much the following characteristics in common:

Are expensive to purchase, implement and maintain.

Are built on proprietary architectures and modeling constructs.

Result quite often in systems that are complex, hard-to-use, difficult to implement and with low user acceptance.

Faced with this, planners and small businesses resort to basic spreadsheets for planning their production environment.

With this project, we are trying to build a light weight tool that can solve simple master planning problems without much hassle. It should allow planners to have access to more advanced functionality while maintaining the flexiblity, ease of use of their original spreadsheets.

For the developer community, the project is also trying to establish some common ground framework for planning applications. Rather than building the basic building blocks (e.g. Items, buffers, boms...) from scratch over and over again, developers can leverage a proven and tested application set to extend. The library is written in C++.

Supported Platforms

The library has been compiled and tested on the following platforms:
  • Windows 2000 and GCC v3.3.3 compiler (Cygwin) but deprecated
  • Windows 2000/XP and GCC v3.4.4 compiler (Cygwin)
  • Windows 2000 and Visual C++ 7.0 compiler
  • Windows 2000 and Intel C++ 9.0 compiler but not tested actively
  • Windows 2000 and Borland 5.5 compiler
  • Linux and GCC v3.4 compiler
The C++ commands and functions are as ANSI compliant as possible, with potentially inportable functions being configured using autoconf. Porting to platforms as well with a modern C++ compiler (i.e. supporting RTTI, namespaces, exceptions, STL, ...) should be relatively easy.

If you get it running successfully on a different platform please let us know, so it can be documented.

Software Prerequisites

No additional software is required to use the executables coming with a binary distribution. No additional software is required also when you want to develop your own applications using the compiled libraries coming with a binary distribution.

Developers building the library from the source code or developing extensions will need to install a number of additional components. These are listed as part of the build instructions.

Model characteristics

Basic building blocks
Have name, category and can be nested in hierarchical trees.
Buffer, item, lot, resource, operation.
Connected by loads and flows.

Higher level building blocks are based upon these.
E.g. Bom, forecast, ...

Problems are flagged when the plan shows infeasibilities or situations requiring user attention.

Subclassing the standard classes allows building more complex and customized models.

Data is currently read in from XML files.
Soon a flat file based input is also expected. Later, a database interface could also be implemented.

Solver characteristics

The model and the solver are logically divided. This makes it possible to use mulitple with the same model. 
The currently application comes with a single solver that is described in more detail below.

The building blocks can be grouped in a number of independent clusters. Each cluster is solved seperately by a multi-threaded solver.

The solver uses a heuristic-based algorithm.
The logic is fully deterministic.
The solver is based on common sense planning logic rather than some mathematical optimization construct.
These features result in a simple, easy to undertand and easy to extend solution. The plan will make sense to the planner, but is not necessarily a mathematical optimum.

In the future, a second solver based on linear programming is expected to be added. This will allow to generate mathematically optimal solutions for subparts of the planning problem.

The algorithm is able to generate a number of different plan types:
- material constrained plan
- lead time constrained plan
- capacity constrained plan

- fully unconstrained plan, a.k.a. MRP run.

- plans with multiple of the basic constraints enabled, e.g. Material and leadtime constrained.

- fully constrained plan, i.e. Constrained by material, leadtime and capacity.
