FREPPLE
a Free Production Planning Library

Plenty of planning tools are available for various production problems, different subprocess, different solver techniques.... A bit suprising, no proper solutions exist in the open source domain. Plenty of users are left with expensive and inflexible tools, and are resorting to spreadsheets to meet their needs.

This project aims at building a lightweight framework that easily and quickly delivers a solution for production planning problems. The initial focus is on small to medium sized master planning problems for the discrete manufacturing industry.

For the developer community, the project is also trying to establish a common ground framework for planning applications. Rather than building the basic building blocks (e.g. item, buffer, bill of material, ...) from scratch over and over again, developers can leverage a proven framweork to extend with their own extension modules.

The word "free" in the project name refers to liberty, not price. Think of "freedomof speech" rather than "free beer": see also the free software definition.

The software is currently in a early-beta stage. This means the software isn't suitable for end users yet. There's plenty of work on the code and the documentation.
I released at this early stage to collect feedback and to gather some interested souls to contribute to the project.

Supported Platforms

The primary platform is linux, using the gcc compiler.

The library can also be compiled on windows (tested on windows 2000 and windows xp), using the Microsoft Visual Studio, the Borland 5.5 or the Intel C++ compiler. The windows version currently offers less functionality. Specifically, multi-threading and dynamically loaded modules aren't supported on this platform.

The C++ commands and functions are as ANSI compliant as possible and porting to various UNIX flavors shouldn't cause too much of an effort. The developer documentation has a section on porting the software.

Software Prerequisites

No additional software is required to use the executables coming with a binary distribution. No additional software is required also when you want to develop your own applications using the compiled libraries coming with a binary distribution.

Model characteristics

Basic building blocks
Have name, category and can be nested in hierarchical trees.
Buffer, item, lot, resource, operation.
Connected by loads and flows.

Higher level building blocks are based upon these.
E.g. Bom, forecast, ...

Problems are flagged when the plan shows infeasibilities or situations requiring user attention.

Subclassing the standard classes allows building more complex and customized models.

Data is currently read in from XML files.
Soon a flat file based input is also expected. Later, a database interface could also be implemented.

Solver characteristics

The model and the solver are logically divided. This makes it possible to use mulitple with the same model. 
The currently application comes with a single solver that is described in more detail below.

The building blocks can be grouped in a number of independent clusters. Each cluster is solved seperately by a multi-threaded solver.

The solver uses a heuristic-based algorithm.
The logic is fully deterministic.
The solver is based on common sense planning logic rather than some mathematical optimization construct.
These features result in a simple, easy to undertand and easy to extend solution. The plan will make sense to the planner, but is not necessarily a mathematical optimum.

In the future, a second solver based on linear programming is expected to be added. This will allow to generate mathematically optimal solutions for subparts of the planning problem.

The algorithm is able to generate a number of different plan types:
- material constrained plan
- lead time constrained plan
- capacity constrained plan

- fully unconstrained plan, a.k.a. MRP run.

- plans with multiple of the basic constraints enabled, e.g. Material and leadtime constrained.

- fully constrained plan, i.e. Constrained by material, leadtime and capacity.
